## 简介
本项目基于React + dnd-kit开发了一套企业官网可视化拖拽编辑器，支持组件拖拽布局配置、实时预览、响应式布局等功能。编辑器采用前后端分离架构，前端使用React + TypeScript + Ant Design，后端提供RESTful API，实现了完整的页面可视化编辑功能。该编辑器基于dnd-kit库实现了高性能的拖拽交互，支持多种组件类型的拖拽、排序和属性配置，现已重构为支持完整的按钮配置系统和容器嵌套功能。

## 核心组件
基于实际实现，拖拽编辑器包含以下核心组件：

### 拖拽布局配置系统
- **页面布局组件库**：预置8种布局模板（hero、text、image、button、container、form、cards、divider、articles）
- **组件拖拽系统**：基于dnd-kit实现的拖拽排序功能
- **实时预览功能**：编辑模式与预览模式无缝切换
- **响应式布局**：支持flexbox和grid布局

### 容器嵌套系统
- **嵌套深度控制**：支持最多2层嵌套的容器结构
- **动态选项过滤**：根据嵌套深度动态显示可选子组件
- **嵌套容器编辑**：支持嵌套容器的属性配置和子组件管理
- **深度计算算法**：递归计算组件在容器树中的嵌套层级

### 子组件尺寸统一控制
- **统一高度设置**：容器级别的childHeight配置，支持80px-400px范围
- **子组件高度控制**：自动应用到所有子组件，保持视觉一致性
- **溢出处理**：超出高度部分自动隐藏，避免布局错乱

### 图片宽高比控制
- **预设比例支持**：1:1、4:3、16:9、3:4四种标准比例
- **padding-top技巧**：使用CSS技巧实现精确的宽高比
- **响应式适配**：自动适配容器宽度，保持比例不变形

### 分类和标签筛选系统
- **动态加载**：支持实时加载分类和标签列表
- **文章组件集成**：文章列表组件支持按分类和标签筛选
- **搜索过滤**：支持分类和标签的搜索和过滤功能

### 组件面板系统
- **组件库面板**：左侧固定宽度，包含所有可用组件
- **画布区域**：中间弹性宽度，支持拖拽操作
- **属性配置面板**：右侧固定宽度，支持组件属性编辑

### 模板系统
- **模板选择器**：提供4种预设模板（默认、企业、科技、服务）
- **模板克隆**：支持模板组件的克隆和个性化定制
- **模板预览**：实时预览模板效果

### 按钮配置系统
- **图标选择**：支持32种预设图标（方向、导航、通讯、媒体、情感、用户、商务、系统、状态）
- **按钮类型**：支持5种样式类型（主要、默认、虚线、文字、链接）
- **尺寸配置**：支持小、中、大三种尺寸
- **颜色自定义**：支持背景色、文字色、边框色的独立配置
- **链接配置**：支持首页、站内页面、外部链接等多种链接类型

### 图片宽高比控制系统

#### 宽高比映射表
系统支持四种标准宽高比，使用padding-top技巧实现精确控制：

| 宽高比 | 比例值 | padding-top百分比 |
|--------|--------|-------------------|
| 1:1 | 1 | 100% |
| 4:3 | 1.3333 | 75% |
| 16:9 | 1.7778 | 56.25% |
| 3:4 | 0.75 | 133.33% |

#### CSS实现技巧
使用padding-top百分比实现固定宽高比的核心原理：

```css
.aspect-ratio-container {
  width: 100%;
  position: relative;
  /* padding-top = (height/width) * 100% */
}

.aspect-ratio-container img {
  position: absolute;
  top: 0;
  left: 0;
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

#### 响应式适配
- **容器自适应**：随容器宽度变化自动调整高度
- **比例保持**：始终保持设定的宽高比
- **内容填充**：图片内容自动填充整个容器

## 故障排除指南

### 常见拖拽问题
1. **拖拽不响应**
   - 检查dnd-kit依赖是否正确安装
   - 验证组件是否正确包裹在DndContext中
   - 确认SortableItem的id属性唯一性

2. **拖拽位置异常**
   - 检查CSS Transform样式是否正确应用
   - 验证组件的ref和setNodeRef绑定
   - 确认z-index层级设置

3. **性能问题**
   - 实施组件状态优化，避免不必要的重渲染
   - 使用React.memo包装渲染器组件
   - 减少深层嵌套的组件结构

### 容器嵌套问题
1. **嵌套容器不可用**
   - 检查嵌套深度计算函数是否正确执行
   - 验证getContainerDepth函数的递归逻辑
   - 确认嵌套容器的props.children结构

2. **子组件添加失败**
   - 检查组件类型是否在可选列表中
   - 验证组件ID生成是否唯一
   - 确认span属性的默认值设置

3. **嵌套折叠异常**
   - 检查Collapse组件的状态管理
   - 验证嵌套容器的展开/折叠逻辑
   - 确认嵌套深度的正确传递

### 尺寸统一控制问题
1. **统一高度不生效**
   - 检查childHeight属性的类型和值
   - 验证CSS高度样式的应用
   - 确认overflow隐藏的设置

2. **高度范围异常**
   - 检查高度滑块的min/max值设置
   - 验证高度值的有效性检查
   - 确认自动转换逻辑的正确性

### 图片宽高比问题
1. **宽高比设置无效**
   - 检查aspectRatio属性的映射表
   - 验证padding-top百分比的计算
   - 确认CSS技巧的应用

2. **图片变形问题**
   - 检查objectFit属性的设置
   - 验证容器尺寸的正确性
   - 确认响应式适配的实现

### 分类标签筛选问题
1. **分类标签加载失败**
   - 检查API请求的正确性
   - 验证数据格式的兼容性
   - 确认错误处理机制

2. **筛选功能异常**
   - 检查分类标签ID的匹配
   - 验证搜索过滤的实现
   - 确认动态选项的更新

### 状态管理问题
- **状态同步**：确保编辑器状态与localStorage同步
- **数据一致性**：验证组件数据结构的完整性
- **回滚机制**：实现撤销/重做功能

### 模板系统问题
- **模板应用**：检查模板克隆函数的ID生成
- **模板预览**：验证模板组件的正确渲染
- **模板切换**：确保模板切换时的数据清理

- **样式应用**：验证buildButtonStyle函数的颜色和尺寸设置
- **链接跳转**：确认getButtonLinkUrl函数的URL生成逻辑
