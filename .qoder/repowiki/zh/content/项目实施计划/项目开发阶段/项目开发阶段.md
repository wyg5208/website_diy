- 修复测试发现的问题
- 性能优化
- 代码清理

**部署步骤**:
1. **Windows Server配置**
   ```bash
   # 创建目录
   mkdir D:\cms
   mkdir D:\cms\data
   mkdir D:\cms\media
   mkdir D:\cms\logs
   ```

2. **安装依赖**
   ```bash
   cd D:\cms\backend
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **数据库初始化**
   ```bash
   flask db upgrade
   python init_data.py  # 创建管理员账号
   ```

4. **配置Nginx**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       # 静态资源
       location /static/ {
           alias D:/cms/frontend/dist/;
       }
       
       # API代理
       location /api/ {
           proxy_pass http://127.0.0.1:5000;
       }
       
       # 前端页面
       location / {
           root D:/cms/frontend/dist;
           try_files $uri /index.html;
       }
   }
   ```

5. **启动服务**
   ```bash
   # 使用Waitress启动（Windows友好）
   waitress-serve --host=127.0.0.1 --port=5000 run:app
   
   # 或使用NSSM注册为Windows服务
   nssm install CMSApp "D:\cms\backend\venv\Scripts\python.exe"
   nssm set CMSApp AppDirectory "D:\cms\backend"
   nssm set CMSApp AppParameters "run.py"
   nssm start CMSApp
   ```

6. **前端构建**
   ```bash
   cd D:\cms\frontend
   npm run build
   # 生成的dist文件夹部署到Nginx
   ```

## 风险管理

### 时间风险
**风险**: 8天周期非常紧凑，任何延误都会影响交付

**应对措施**:
1. 严格采用MVP策略，只做核心功能
2. 每日站会，及时发现和解决问题
3. 并行开发，后端和前端同步进行
4. 预留缓冲，第11日留足测试和修复时间
5. 加班预案，关键节点可适当加班

### 技术风险

**风险1: 可视化编辑器开发复杂**
- **应对**: 使用成熟的拖拽库（react-dnd-kit）
- **简化方案**: 只做5个基础组件，不做复杂嵌套

**风险2: Windows环境部署问题**
- **应对**: 使用Waitress而非Gunicorn
- **提前准备**: 第1天就配置好Windows环境

**风险3: SQLite并发写入限制**
- **应对**: 启用WAL模式，优化写入性能
- **场景限制**: MVP阶段用户少，不会有高并发写入

### 人员风险
**风险**: 团队成员生病或其他突发情况

**应对措施**:
1. 关键模块至少2人熟悉
2. 代码每日提交，确保进度可见
3. 准备技术备份人员
