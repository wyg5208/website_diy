## 项目背景

企业网站CMS系统旨在解决传统企业官网建设中的痛点问题。当前市场上的企业官网存在以下挑战：

- **技术门槛高**：需要专业技术人员才能维护网站内容
- **更新效率低**：内容变更流程繁琐，响应速度慢
- **成本高昂**：定制开发费用昂贵，后期维护成本高
- **功能单一**：缺乏可视化编辑和多终端适配能力

针对这些痛点，本项目致力于打造一套功能完善、易于维护的企业官网内容管理系统，通过可视化拖拽配置降低技术门槛，提升网站管理效率。

**更新** 项目已从非Git项目迁移到Git仓库，建立了完整的Flask/React全栈架构，包括JWT认证、数据库设计、前后端分离等完整技术栈。

## 项目目标与价值主张

### 核心目标

1. **快速搭建企业官网**：实现企业官网的快速搭建和灵活配置
2. **直观可视化编辑**：提供拖拽式的可视化编辑体验
3. **多终端适配**：支持PC、平板、手机等多终端访问
4. **SEO优化支持**：内置SEO优化功能，提升搜索引擎排名
5. **安全可靠**：确保系统安全性和可扩展性

### 价值主张

- **降低运营成本**：非技术人员可独立完成网站内容更新
- **提升管理效率**：可视化操作替代技术开发，提高工作效率
- **增强用户体验**：多终端适配和响应式设计
- **保障数据安全**：完善的权限管理和安全防护机制
- **支持多站点管理**：一套系统管理多个企业站点

## 系统架构理念

### 整体架构模式

系统采用**前后端分离架构**，结合**混合模式支持**的设计理念：

```mermaid

end

Browser --> React
Mobile --> React
AdminPanel --> React
React --> Flask
React --> JWT
JWTAuth --> Flask
Flask --> SQLite
Flask --> Media
Flask --> Logs
```

**架构优势**：
- **高可用性**：前后端分离，职责清晰，便于维护和扩展
- **安全性**：JWT Token认证，CORS跨域控制，多层安全防护
- **可扩展性**：模块化设计，支持微服务化改造
- **易维护性**：清晰的前后端边界，独立开发和部署

### 技术栈选择

**后端技术栈**：
- **Python Flask**：轻量级Web框架，开发效率高
- **Flask-JWT-Extended**：JWT认证扩展，提供安全的Token管理
- **SQLAlchemy**：ORM对象关系映射，简化数据库操作
- **Flask-Migrate**：数据库迁移工具，支持版本管理
- **Flask-CORS**：跨域资源共享，支持前后端分离

**前端技术栈**：
- **React 19**：现代JavaScript框架，组件化开发
- **TypeScript**：类型安全，提升代码质量
- **Ant Design**：企业级UI组件库
- **Vite**：快速构建工具，开发体验优秀
- **Axios**：HTTP客户端，支持请求拦截器

**更新** 项目已建立完整的Flask/React技术栈，包括JWT认证、数据库设计、前端组件化开发等完整架构。

## 技术选型与优势

### 技术架构优势

**1. 轻量级部署**
- SQLite3单文件数据库，零配置部署
- 无需专门的数据库服务，简化运维
- 支持Windows Server环境，降低部署成本

**2. 高性能设计**
- 前后端分离架构，独立优化
- JWT Token认证，减少数据库查询
- 前端构建优化，减少资源加载

**3. 安全性保障**
- JWT Token认证机制
- XSS、CSRF、SQL注入防护
- 文件上传安全验证和病毒扫描

**4. 可扩展性**
- 模块化架构设计
- 支持插件扩展
- 微服务化改造预留

### 技术栈对比分析

| 技术栈 | 优势 | 适用场景 | 风险 |
|--------|------|----------|------|
| Python Flask + SQLite3 | 部署简单、成本低、开发效率高 | 中小型企业官网 | 数据量大时性能瓶颈 |
| React/Vue.js | 组件化开发、生态丰富、开发体验好 | 现代Web应用 | 学习曲线较陡峭 |
| Nginx | 高性能、稳定、配置灵活 | 生产环境部署 | 配置复杂度较高 |
| Redis | 缓存性能优异、功能丰富 | 高并发场景 | 内存成本较高 |

**更新** 项目已采用最新的Flask 3.1.2和React 19技术栈，提供了更好的性能和开发体验。
