# 页面编辑功能使用指南

## 概述

本系统支持对"解决方案"、"成功案例"、"关于我们"三个页面进行可视化拖拽编辑，实现所见即所得的设计体验。

## 如何访问页面编辑器

### 方法一：通过后台管理
1. 登录后台管理系统 (http://localhost:5173/login)
2. 点击左侧菜单 "页面管理" → "页面编辑器"
3. 在页面选择下拉框中选择要编辑的页面

### 方法二：从前台页面直接编辑
1. 访问对应的前台页面：
   - 解决方案：http://localhost:5173/solutions
   - 成功案例：http://localhost:5173/cases  
   - 关于我们：http://localhost:5173/about
   - 自定义页面：http://localhost:5173/[页面key]
2. 如果页面为空，会显示"立即编辑此页面"按钮（需先登录）

## 编辑器功能介绍

### 1. 页面选择
- 使用顶部下拉框切换不同的页面进行编辑
- 支持：首页、解决方案、成功案例、关于我们

### 2. 预设模板
针对三个特定页面提供了专门的模板：

**🎯 解决方案模板**（仅在编辑 /solutions 页面时可用）
- 蓝色主题横幅
- 三列服务卡片布局
- 包含：网站开发、移动应用、系统集成等内容

**🏆 成功案例模板**（仅在编辑 /cases 页面时可用）  
- 绿色主题横幅
- 精选案例展示
- 三列案例卡片布局

**👥 关于我们模板**（仅在编辑 /about 页面时可用）
- 紫色主题横幅
- 公司简介图文布局
- 核心优势四列展示

### 3. 组件库
支持以下可拖拽组件：
- **横幅(Hero)** - 页面顶部大图横幅
- **文本(Text)** - 富文本内容块
- **图片(Image)** - 图片展示组件
- **按钮(Button)** - 可点击按钮
- **容器(Container)** - 布局容器，可包含其他组件
- **表单(Form)** - 联系表单
- **卡片列表(Cards)** - 多卡片展示
- **分割线(Divider)** - 页面分割线

### 4. 操作功能
- **拖拽排序** - 直接拖动组件调整顺序
- **属性编辑** - 点击组件后在右侧面板编辑属性
- **预览模式** - 切换编辑/预览模式查看效果
- **保存草稿** - 临时保存到本地
- **发布上线** - 保存到服务器并立即生效

## 使用步骤

### 快速开始（使用模板）
1. 进入页面编辑器
2. 选择目标页面（solutions/cases/about）
3. 点击对应页面的预设模板按钮
4. 根据需要调整组件内容和样式
5. 点击"发布上线"保存

### 自定义设计
1. 进入页面编辑器
2. 从左侧组件库拖拽组件到画布
3. 点击组件进行选中
4. 在右侧属性面板调整组件参数
5. 重复步骤2-4添加更多组件
6. 使用拖拽调整组件顺序
7. 点击"发布上线"保存

## 编辑技巧

### 容器组件使用
- 容器可以包含其他组件，实现复杂布局
- 支持垂直、水平、网格三种布局模式
- 可设置背景色、内边距、最大宽度等样式

### 响应式设计
- 系统自动适配不同屏幕尺寸
- 卡片组件支持响应式列数调整
- 图片组件支持多种填充模式

### 内容管理
- 文本组件支持富文本编辑
- 图片组件可设置外部图片链接
- 表单组件可自定义字段

## 注意事项

1. **保存机制**：建议编辑过程中定期点击"保存草稿"，避免意外丢失
2. **发布生效**：只有点击"发布上线"后，前台页面才会显示最新内容
3. **权限要求**：编辑页面需要登录管理员账号
4. **浏览器兼容**：建议使用最新版Chrome、Firefox或Edge浏览器
5. **自定义页面访问**：通过菜单管理创建的自定义页面，访问路径为 `http://localhost:5173/[页面key]`

## 常见问题

**Q: 编辑后前台页面没有变化？**
A: 请确保点击了"发布上线"按钮，草稿仅保存在本地。

**Q: 如何恢复之前的版本？**
A: 系统暂不支持版本回滚，建议重要修改前先导出页面配置。

**Q: 模板内容不满意怎么办？**
A: 可以清空画布后重新设计，或在模板基础上进行修改。

**Q: 组件样式如何调整？**
A: 选中组件后，在右侧属性面板中调整各项参数。

**Q: 通过菜单管理创建的页面为什么前台访问不到？**
A: 确保：
1. 已在页面编辑器中为该页面key编辑并发布了内容
2. 访问路径正确：`http://localhost:5173/[页面key]`
3. 页面key不与系统固定路由冲突（如solutions、cases、about等）
