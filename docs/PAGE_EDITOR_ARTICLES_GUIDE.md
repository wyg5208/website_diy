# 页面编辑器文章组件使用指南

## 🎯 功能概述

现在你可以在后台的**页面编辑器**中添加"文章列表"组件，配置后前台页面会自动获取并显示文章。

---

## 📝 使用步骤

### 步骤 1：进入页面编辑器

1. 访问后台：`http://localhost:5173/admin`
2. 点击左侧菜单：**页面管理** → **页面编辑**
3. 选择要编辑的页面（如：网站首页、解决方案等）

### 步骤 2：添加文章列表组件

1. 在左侧组件面板中找到 **"📄 文章列表"**
2. 点击按钮添加到画布中
3. 文章列表组件会立即显示在页面上

### 步骤 3：配置组件属性

在右侧属性面板中配置：

#### 基础配置

| 属性 | 说明 | 默认值 |
|------|------|--------|
| 显示模式 | 列表模式 / 网格模式 | 列表模式 |
| 显示标题区域 | 是否显示组件标题 | 否 |
| 自定义标题 | 当显示标题时，自定义标题文字 | 最新文章 |
| 每页显示数量 | 一次显示多少篇文章 | 10 |
| 显示分页 | 是否显示分页控件 | 是 |

#### 筛选配置

| 属性 | 说明 | 示例 |
|------|------|------|
| 按分类筛选 | 输入分类 ID，留空显示所有分类 | `1`（公司新闻） |
| 按标签筛选 | 输入标签名称或 ID，留空显示所有标签 | `技术分享` |

**如何获取分类 ID？**
- 方法 1：后台 → 分类管理，查看 URL 或浏览器开发者工具
- 方法 2：数据库查询：`SELECT * FROM categories;`
- 方法 3：调用 API：`GET /api/v1/categories`

### 步骤 4：保存并发布

1. 点击右上角 **"保存草稿"**（本地保存）
2. 确认无误后点击 **"发布上线"**（同步到服务器）
3. 访问前台页面查看效果

## 💡 使用场景

### 场景 1：首页显示公司新闻

1. 编辑首页（`/admin/pages/editor?page=home`）
2. 添加文章列表组件
3. 配置：
   - 显示模式：列表
   - 按分类筛选：`1`（假设"公司新闻"ID 为 1）
   - 每页显示：`5`
   - 显示分页：否
4. 发布

**效果**：首页自动显示最新 5 条公司新闻

---

### 场景 2：解决方案页面显示案例文章

1. 编辑解决方案页面
2. 添加文章列表组件
3. 配置：
   - 显示模式：网格
   - 显示标题：是
   - 自定义标题：成功案例
   - 按标签筛选：`案例`
4. 发布

**效果**：以网格形式展示所有标记为"案例"的文章

---

### 场景 3：技术博客页面

1. 创建新页面（如：`blog`）
2. 添加文章列表组件
3. 配置：
   - 显示模式：列表
   - 显示标题：是
   - 自定义标题：技术博客
   - 按分类筛选：`2`（假设"技术分享"ID 为 2）
   - 每页显示：`10`
   - 显示分页：是
4. 发布

**效果**：完整的技术博客列表页，支持分页

## 🔧 高级配置

### 组合使用多个文章列表

同一个页面可以添加多个文章列表组件，实现更丰富的展示：

**示例：首页布局**
```
1. Hero 横幅（公司标语）
2. 文章列表 1：公司新闻（列表模式，显示 3 条）
3. 容器（两列布局）
   - 左列：文章列表 2：行业动态（显示 5 条）
   - 右列：文章列表 3：技术文章（显示 5 条）
4. 文章列表 4：所有文章（网格模式，带分页）
```

### 与其他组件搭配

文章列表可以与以下组件完美搭配：

- **Hero 横幅**：作为页面开头，引导到文章列表
- **文本块**：在文章列表上方添加说明文字
- **分割线**：分隔不同的文章列表
- **容器**：创建多列布局，并排显示不同类别的文章

## 🐛 常见问题

### Q1: 组件显示"暂无文章"？

**A**: 可能原因：
1. 还没有发布任何文章
2. 筛选条件太严格（分类/标签不匹配）
3. 文章状态不是"已发布"

**解决方案**：
- 检查后台是否有文章：`/admin/posts`
- 确保文章状态为"已发布"
- 尝试清空筛选条件（分类和标签留空）

---

### Q2: 如何知道分类 ID？

**A**: 三种方法：

**方法 1：后台查看**
1. 访问后台分类管理
2. 鼠标悬停在分类上
3. 查看浏览器状态栏或开发者工具

**方法 2：API 查询**
```bash
GET http://localhost:5000/api/v1/categories
```

**方法 3：数据库查询**
```sql
SELECT id, name FROM categories;
```

---

### Q3: 能否显示单篇指定文章？

**A**: 目前不支持。文章列表组件设计用于显示多篇文章。

如需显示单篇文章，可以使用：
- **文本块组件**：手动输入文章内容
- **自定义页面**：为特定文章创建专用页面

---

### Q4: 文章更新后，页面会自动更新吗？

**A**: 是的！

文章列表组件每次加载时都会从后端获取最新文章数据，所以：
- ✅ 发布新文章 → 页面自动显示
- ✅ 修改文章 → 页面自动更新
- ✅ 删除文章 → 页面自动移除

无需重新编辑页面！

## 📊 数据结构

```json

  "id": "uuid-1234",
  "type": "articles",
  "props": {
    "displayMode": "list",
    "showTitle": true,

    "limit": 10,
    "showPagination": true,
```

## 🚀 性能优化建议

### 1. 合理设置显示数量
- 首页/侧边栏：3-5 条
- 列表页：10-20 条
- 避免一次性显示过多文章

### 2. 启用分页
- 文章总数 > 显示数量时，建议开启分页
- 减少单次加载数据量

### 3. 使用筛选
- 通过分类/标签缩小范围
- 提高加载速度和用户体验

---

## 🎯 最佳实践

### 1. 保持一致性
- 同一页面尽量使用一种显示模式
- 多篇章列表时，保持样式统一

### 2. 标题清晰
- 使用有意义的标题（如"公司新闻"而非"文章列表"）
- 帮助用户快速理解内容

### 3. 适度使用
- 一个页面 1-3 个文章列表为宜
- 避免过多导致页面冗长

### 4. 移动端适配
- 网格模式在移动端自动变为单列
- 列表模式天然适合移动端
