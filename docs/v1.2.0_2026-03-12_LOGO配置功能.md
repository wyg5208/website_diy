## 🎯 迭代目标

在企业网站 CMS 系统的后台管理中添加 LOGO 配置功能，支持管理员自定义网站左上角的 LOGO 展示，包括：
- 文字 + 图片组合显示
- 仅文字显示
- 仅图片显示
- 可配置文字大小、颜色、字体粗细等样式
- 可配置图片尺寸和间距

#### 新增接口

**获取 LOGO 配置**
- **URL**: `/api/v1/settings/logo`
- **方法**: GET
- **认证**: 无需认证（公开接口）
- **功能**: 获取网站 LOGO 配置信息
- **默认配置**: 当数据库中无配置时返回预设默认值

**更新 LOGO 配置**
- **URL**: `/api/v1/settings/logo`
- **方法**: PUT
- **认证**: 需要 JWT 认证
- **功能**: 保存 LOGO 配置信息
- **验证**: 必需字段 `enabled` 和 `displayMode`

#### 数据结构

```typescript
interface LogoConfig {
  enabled: boolean;                    // 是否启用
  displayMode: 'textOnly' | 'imageOnly' | 'textAndImage';  // 显示模式
  logoImage?: string;                  // LOGO 图片 URL
  logoImageWidth: number;              // 图片宽度 (px)
  logoImageHeight: number;             // 图片高度 (px)
  logoText: string;                    // 主文字
  logoSubText?: string;                // 副标题文字
  textColor: string;                   // 主文字颜色
  subTextColor?: string;               // 副标题颜色
  fontSize: number;                    // 主文字大小 (px)
  subFontSize?: number;                // 副标题大小 (px)
  fontWeight: number;                  // 文字粗细 (400/600/800)
  letterSpacing: number;               // 文字间距
  imageGap?: number;                   // 图片与文字间距 (px)
  linkUrl: string;                     // 点击跳转链接
}
```

```typescript
{logoConfig && logoConfig.enabled ? (
  // 自定义 LOGO
  <Link to={logoConfig.linkUrl}>
    <div style={{ display: 'flex', alignItems: 'center' }}>
      {/* 图片部分 */}
      {(logoConfig.displayMode === 'textAndImage' || 
        logoConfig.displayMode === 'imageOnly') && logoConfig.logoImage && (
        <img
          src={logoConfig.logoImage}
          alt="LOGO"
          style={{
            width: `${logoConfig.logoImageWidth}px`,
            height: `${logoConfig.logoImageHeight}px`,
            objectFit: 'contain',
            marginRight: logoConfig.displayMode === 'textAndImage' 
              ? `${logoConfig.imageGap}px` 
              : 0
          }}
        />
      )}

      {/* 文字部分 */}
      {(logoConfig.displayMode === 'textAndImage' || 
        logoConfig.displayMode === 'textOnly') && (
        <div style={{ display: 'flex', alignItems: 'baseline' }}>
          <span style={{
            fontSize: `${logoConfig.fontSize}px`,
            color: logoConfig.textColor,
            fontWeight: logoConfig.fontWeight,
            letterSpacing: `${logoConfig.letterSpacing}px`
          }}>
            {logoConfig.logoText}
          </span>
          {logoConfig.logoSubText && (
            <span style={{
              fontSize: `${logoConfig.subFontSize}px`,
              color: logoConfig.subTextColor,
              fontWeight: logoConfig.fontWeight,
              marginLeft: '2px'
            }}>
              {logoConfig.logoSubText}
            </span>
          )}
        </div>
      )}
    </div>
  </Link>
) : (
  // 默认 LOGO
  <Link to="/">
    <div style={{ 
      color: colors.primary, 
      fontSize: '22px', 
      fontWeight: '800', 
      marginRight: '60px', 
      letterSpacing: '1px' 
    }}>
      CORP<span style={{ color: colors.textPrimary }}>CMS</span>
    </div>
  </Link>
)}
```

## 🐛 问题修复记录

### 问题 1: 模块导出错误
**错误信息**: 
```
The requested module '/src/api/posts.ts' does not provide an export named 'getMediaList'
```

**原因**: `LogoConfig.tsx` 尝试从不存在的 `posts.ts` 导入 `getMediaList`

**解决方案**:
1. 创建独立的 `media.ts` API 文件
2. 修正导入路径为 `../api/media`

### 问题 2: 类型访问错误
**错误信息**: 
```
Cannot read properties of undefined (reading 'length')
```

**原因**: `Home.tsx` 中 `posts` 变量可能是 `undefined`

**解决方案**:
1. 数据加载时使用可选链：`postsRes.data?.items || []`
2. 异常处理中确保设置为空数组：`setPosts([])`
3. 渲染时使用安全检查：`(posts?.length ?? 0) === 0`

### 问题 3: API 响应结构不匹配
**错误信息**: 多个 TypeScript 类型错误

**原因**: Axios 响应对象结构理解有误，应该访问 `response.data.code` 而非 `response.code`

**解决方案**:
- 统一修改为 `response.data.code`、`response.data.data`、`response.data.message`
- 在 `LogoConfig.tsx` 和 `Home.tsx` 中修正所有响应数据访问
