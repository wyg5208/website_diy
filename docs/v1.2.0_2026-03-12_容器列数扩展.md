## 需求背景

用户反馈：页面编辑器中容器组件的子组件布局，原有列数选项有限，无法满足更复杂的多列布局需求，希望支持更多列数配置。

---

## 变更内容

### 1. 容器组件 (Container) 列数扩展

**修改前：** 支持 1、2、3、4、6 列

**修改后：** 支持 1、2、3、4、5、6、8、10、12 列

### 2. 卡片列表组件 (Cards) 列数扩展

**修改前：** 每行最多 4 列 (`max=4`)

**修改后：** 每行最多 12 列 (`max=12`)

---

| 文件路径 | 修改说明 |

| `frontend/src/pages/PageEditor.tsx` | 扩展容器和卡片组件的列数配置选项 |

## 代码变更详情

### PageEditor.tsx - 容器列数选择器

```diff
  <Select value={props.columns || 3} onChange={(v) => handlePropsChange('columns', v)} options={[
    { value: 1, label: '1 列' },
    { value: 2, label: '2 列' },
    { value: 3, label: '3 列' },
    { value: 4, label: '4 列' },
+   { value: 5, label: '5 列' },
    { value: 6, label: '6 列' },
+   { value: 8, label: '8 列' },
+   { value: 10, label: '10 列' },
+   { value: 12, label: '12 列' },
  ]} />
```

### PageEditor.tsx - 卡片列数限制

```diff
- <InputNumber min={1} max={4} value={props.columns} ... />
+ <InputNumber min={1} max={12} value={props.columns} ... />
```

## 测试要点

- [ ] 容器组件在网格布局模式下，验证 5、8、10、12 列显示正常
- [ ] 容器组件在水平布局模式下，验证多列子组件宽度计算正确
- [ ] 卡片列表组件设置超过 4 列时，卡片渲染正常

---

## 备注

本次迭代基于12列网格系统设计，支持常见的响应式布局场景。子组件的 `span` 属性（占据列数）会自动适配容器设置的总列数。
