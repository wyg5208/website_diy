# 迭代记录：文章作者显示问题修复

**日期**: 2026-03-16  
**版本**: v1.3.0  
**类型**: Bug修复 + 代码优化

---

## 问题描述

文章列表和详情页中，作者字段显示为 "???"，影响用户体验。

### 问题现象

```
文章: 测试111 作者: ???
文章: jjj 作者: ???
文章: 啊啊啊啊 作者: ???
...
```

---

## 问题排查

### 1. 问题定位过程

**Step 1: 检查前端代码**

前端使用 `post.author_name` 显示作者名称，代码逻辑正常：

```tsx
// PostList.tsx
{
  title: '作者',
  dataIndex: 'author_name',
  key: 'author_name',
}

// PostDetailPage.tsx
<Text type="secondary">作者：{post.author_name}</Text>
```

**Step 2: 检查后端模型**

Post 模型的 `to_dict()` 方法中：

```python
# post.py (原代码)
'author_name': self.author.display_name or self.author.username,
```

逻辑看似正确，但存在潜在问题：
- 如果 `self.author` 为 None（用户被删除），会抛出 `AttributeError`
- 如果 `display_name` 本身就是无效值，无法自动回退

**Step 3: 检查数据库数据**

```sql
SELECT id, username, display_name FROM users;
```

结果：

| id | username | display_name |
|----|----------|--------------|
| 1 | admin | ??? |
| 2 | testuser | ???? |
| 3 | admin1 | 阿翁 |
| 4 | admin2 | 阿翁2 |

### 2. 根因分析

**数据库中 `display_name` 字段的值本身就是 "???"**

可能原因：
1. **数据库编码问题**：SQLite 在某些情况下可能无法正确存储中文字符
2. **早期数据初始化问题**：初始数据插入时编码配置不正确
3. **前端提交了错误数据**：用户编辑时可能提交了 "???"

---

## 解决方案

### 方案一：数据修复

修复数据库中无效的 `display_name` 值：

```python
from app import create_app, db
from app.models import User

app = create_app()
with app.app_context():
    users = User.query.all()
    for user in users:
        # 检查 display_name 是否只包含问号或为空
        if user.display_name and all(c == '?' for c in user.display_name):
            user.display_name = user.username
        elif not user.display_name:
            user.display_name = user.username
    
    db.session.commit()
```

**修复结果**：

```
修复用户: admin, 原display_name: ??? -> admin
修复用户: testuser, 原display_name: ???? -> testuser
共修复 2 个用户
```

### 方案二：代码优化

优化 `Post.to_dict()` 方法，添加防护逻辑：

**修改文件**: `backend/app/models/post.py`

```python
def to_dict(self):
    """转换为字典"""
    # 安全获取作者名称：处理作者不存在或display_name无效的情况
    author_name = '未知用户'
    if self.author:
        # 检查 display_name 是否有效（非空且不全为问号）
        if self.author.display_name and not all(c == '?' for c in self.author.display_name):
            author_name = self.author.display_name
        else:
            author_name = self.author.username
    
    return {
        'id': self.id,
        'type': self.type,
        'title': self.title,
        'slug': self.slug,
        'content': self.content,
        'content_format': self.content_format,
        'excerpt': self.excerpt,
        'featured_image': self.featured_image,
        'author_id': self.author_id,
        'author_name': author_name,
        # ... 其他字段
    }
```

### 优化点说明

| 场景 | 原逻辑 | 优化后逻辑 |
|------|--------|------------|
| 作者不存在 | 抛出 AttributeError | 显示 "未知用户" |
| display_name 为空 | 回退到 username | 回退到 username |
| display_name 全为问号 | 显示问号 | 回退到 username |
| display_name 正常 | 显示 display_name | 显示 display_name |

## 经验总结

### 1. 数据完整性校验

在用户注册/更新时，应对 `display_name` 进行有效性校验：

```python
# 建议在 User 模型或 API 层添加校验
def validate_display_name(name):
    """校验显示名称有效性"""
    if not name or all(c == '?' for c in name):
        return False
    return True
```

### 2. 外键关联安全访问

在使用外键关联对象时，应始终考虑关联对象不存在的情况：

```python
# 不安全
author_name = self.author.display_name

# 安全
author_name = self.author.display_name if self.author else '未知用户'
```

### 3. 数据库编码规范

- SQLite 默认支持 UTF-8，中文存储应无问题
- 问题可能出在数据初始化脚本或手动插入数据时的编码问题
- 建议使用 ORM 进行数据操作，避免手动 SQL 导致的编码问题
