## 一、修订背景

### 问题描述

1. **媒体库上传 401 认证失败**
   - 用户在使用媒体库上传图片时，出现 `POST /api/v1/media/upload 401 (UNAUTHORIZED)` 错误
   - 原因：antd Upload 组件使用 `action` 属性时，`headers` 配置在组件渲染时就固定了，导致 token 无法动态获取

2. **Token 过期无友好提示**
   - 用户长时间未操作导致 token 过期后，上传失败但没有任何提示
   - 用户不知道需要重新登录，造成困惑

3. **文件类型支持有限**
   - 原来仅支持 `png, jpg, jpeg, gif, webp` 五种图片格式
   - 用户上传 JFIF 等常见格式时被拒绝

#### 核心改动

```typescript

const uploadProps: UploadProps = {
  action: 'http://127.0.0.1:5000/api/v1/media/upload',
  headers: {
    Authorization: `Bearer ${localStorage.getItem('token')}`,  // 渲染时固定
  },
};

// 修改后：customRequest 动态获取 token
const uploadProps: UploadProps = {
  customRequest: async (options) => {
    const token = localStorage.getItem('token');  // 每次上传时获取
    const response = await fetch(url, {
      headers: { 'Authorization': `Bearer ${token}` },
      body: formData,
    });
    // ...
  },
};
```

### 2.2 Token 过期友好提示

#### 修改文件

| 文件路径 | 修改说明 |
|---------|---------|
| `frontend/src/utils/request.ts` | 新增 `handleLoginExpired` 统一处理函数 |

#### 核心改动

```typescript
// 统一的登录过期处理函数
let isShowingLoginExpired = false;
const handleLoginExpired = () => {
  if (isShowingLoginExpired) return; // 防止多次弹窗
  isShowingLoginExpired = true;
  
  localStorage.removeItem('token');
  localStorage.removeItem('user');
  
  message.warning('登录已过期，请重新登录', 3, () => {
    isShowingLoginExpired = false;
    window.location.href = '/login';
  });
};
```

#### 效果

- 收到 401 错误时显示黄色警告提示
- 清除本地存储的 token 和 user 信息
- 3 秒后自动跳转到登录页
- 防止多个请求同时触发多次弹窗

## 四、部署注意事项

1. **后端需要重启**：修改 `config.py` 后需要重启 Flask 服务才能生效

2. **环境变量配置**：生产环境可通过 `ALLOWED_EXTENSIONS` 环境变量自定义支持的文件类型

3. **文件大小限制**：默认 50MB，可通过 `MAX_CONTENT_LENGTH` 环境变量调整
