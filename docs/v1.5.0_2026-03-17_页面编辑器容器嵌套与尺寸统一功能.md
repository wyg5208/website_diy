## 需求背景

### 问题一：容器无法嵌套
用户反馈：页面编辑器中容器只能添加基础组件（文本、图片、按钮等），无法添加嵌套容器，导致无法配置复杂的页面结构。

### 问题二：子组件尺寸不统一
用户反馈：容器中不同尺寸的图片会导致显示效果参差不齐，影响页面美观，希望有统一控制子组件尺寸的能力。

---

## 解决方案

### 功能一：容器嵌套

支持容器内添加嵌套容器，实现复杂页面结构配置：
- 最大嵌套深度限制为 2 层
- 第 1 层容器可以添加嵌套容器
- 第 2 层容器只能添加基础组件，不能再嵌套

### 功能二：子组件尺寸统一控制

**容器级别**：
- 新增 `childHeight` 配置，统一控制所有子组件的高度
- 支持 80px ~ 400px 多档位选择

**图片组件**：
- 新增 `aspectRatio` 宽高比配置
- 支持 1:1、4:3、16:9、3:4 四种预设比例

## 技术亮点

### 1. 嵌套深度计算
```typescript
// 递归计算容器嵌套深度
const getContainerDepth = (componentId: string, componentList: PageComponent[]): number => {
  for (const comp of componentList) {
    if (comp.type === 'container' && comp.props.children) {
      const found = comp.props.children.find((child) => child.id === componentId);
      if (found) return 1;
      // 递归检查嵌套容器
      for (const child of comp.props.children) {
        if (child.type === 'container' && child.props?.children) {
          const depth = getNestedDepth(componentId, child.props.children);
          if (depth > 0) return 1 + depth;
        }
      }
    }
  }
  return 0;
};
```

### 2. 动态选项过滤
根据嵌套深度动态过滤可选子组件类型：
```typescript
const currentDepth = selectedComponent 
  ? getContainerDepth(selectedComponent.id, components) 
  : 0;

const availableChildTypes = currentDepth === 0
  ? [...baseOptions, { value: 'container', label: '容器（嵌套）' }]
  : baseOptions;  // 嵌套容器内不显示容器选项
```

### 3. CSS 宽高比技巧
使用 padding-top 百分比实现固定宽高比，避免图片变形。
