**部署步骤**:
1. **Windows Server配置**
   ```bash
   # 创建目录
   mkdir D:\cms
   mkdir D:\cms\data
   mkdir D:\cms\media
   mkdir D:\cms\logs
   ```

2. **安装依赖**
   ```bash
   cd D:\cms\backend
   python -m venv venv
   venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **数据库初始化**
   ```bash
   flask db upgrade
   python init_data.py  # 创建管理员账号
   ```

4. **配置Nginx**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       # 静态资源
       location /static/ {
           alias D:/cms/frontend/dist/;
       }
       
       # API代理
       location /api/ {
           proxy_pass http://127.0.0.1:5000;
       }
       
       # 前端页面
       location / {
           root D:/cms/frontend/dist;
           try_files $uri /index.html;
       }
   }
   ```

5. **启动服务**
   ```bash
   # 使用Waitress启动(Windows友好)
   waitress-serve --host=127.0.0.1 --port=5000 run:app
   
   # 或使用NSSM注册为Windows服务
   nssm install CMSApp "D:\cms\backend\venv\Scripts\python.exe"
   nssm set CMSApp AppDirectory "D:\cms\backend"
   nssm set CMSApp AppParameters "run.py"
   nssm start CMSApp
   ```

6. **前端构建**
   ```bash
   cd D:\cms\frontend
   npm run build
   # 生成的dist文件夹部署到Nginx
   ```

## 四、风险管理

### 4.1 时间风险

**风险**: 8天周期非常紧凑,任何延误都会影响交付

**应对措施**:
1. **严格采用MVP策略**,只做核心功能
2. **每日站会**,及时发现和解决问题
3. **并行开发**,后端和前端同步进行
4. **预留缓冲**,第11日留足测试和修复时间
5. **加班预案**,关键节点可适当加班

### 4.2 技术风险

**风险1: 可视化编辑器开发复杂**
- **应对**: 使用成熟的拖拽库(react-dnd-kit)
- **简化方案**: 只做5个基础组件,不做复杂嵌套

**风险2: Windows环境部署问题**
- **应对**: 使用Waitress而非Gunicorn
- **提前准备**: 第1天就配置好Windows环境

**风险3: SQLite并发写入限制**
- **应对**: 启用WAL模式,优化写入性能
- **场景限制**: MVP阶段用户少,不会有高并发写入

### 4.3 人员风险

**风险**: 团队成员生病或其他突发情况

**应对措施**:
1. 关键模块至少2人熟悉
2. 代码每日提交,确保进度可见
3. 准备技术备份人员

**应对措施**:
1. 立即召开紧急会议
2. 砍掉非核心功能(如:分类管理、系统设置)
3. 简化可视化编辑器(只保留文本和图片组件)

1. 改用成熟的编辑器方案(如TinyMCE)
2. 放弃拖拽功能,改用传统表单编辑

**应对措施**:
1. 使用Docker容器化部署

3. 提前准备备用方案

## 附录: 快速启动指南

### A1. 后端快速启动

```bash
# 克隆代码
git clone <repository>
cd backend

# 创建虚拟环境
python -m venv venv
venv\Scripts\activate  # Windows
source venv/bin/activate  # Linux/Mac

# 安装依赖
pip install -r requirements.txt

# 初始化数据库
flask db upgrade
python init_data.py

# 启动开发服务器
flask run
# 访问: http://localhost:5000
```

### A2. 前端快速启动

```bash
# 安装依赖
cd frontend
npm install

# 启动开发服务器
npm run dev
# 访问: http://localhost:3000

# 生产构建
npm run build
```
